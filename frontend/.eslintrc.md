# ESLint Configuration for GreenPulse Frontend

Enforces React, TypeScript, and code quality best practices.

## Rules Enabled

- ✓ React best practices
- ✓ React Hooks rules
- ✓ TypeScript strict checking
- ✓ Unused variables detection
- ✓ Merge conflict detection

## Usage

```bash
# Check all files
npm run lint

# Fix auto-fixable issues
npm run lint -- --fix

# Check specific file
npm run lint src/App.tsx
```

## Disable Rules (When Necessary)

Add ESLint comment (use sparingly):
```tsx
// eslint-disable-next-line react-hooks/exhaustive-deps
```
